# 象棋 AI 分析講解 v0.3.2

v0.3.2 修正 AI 教練實戰步與 AI 首選比較在引擎證據完整時仍回覆「證據不足」的問題，並完成內建 Pikafish 與 GitHub Release 全新安裝流程。

## 主要改善

- 將 Gemini 單次解說等待時間調整為符合實際推理延遲，避免 App 在模型完成前自行中止。
- 將使用者輸入的走棋理由連同完整引擎主線交給模型驗證，讓「實戰步」與「AI 首選」能解釋具體戰略差異。
- 模型逾時或 Provider 錯誤時會如實指出 AI 服務未完成，不再誤稱 Pikafish「引擎證據不足」。
- 安裝版內附 Pikafish Windows AVX2 執行檔、NNUE 權重與完整第三方授權／來源說明；乾淨安裝會自動登錄內建引擎，不必再手動尋找 EXE。
- 自動更新與手動下載統一使用 `enzohuang98-crypto/Reckoning` 的 GitHub Releases，解除舊 Git-backed 更新網站的 100 MiB 限制。
- 補齊 README、授權、貢獻與安全政策、Issue／PR 模板、Dependabot、Windows CI 與分支保護等公開專案基本設定；未改動象棋搜尋、評分或棋盤核心演算法。
- 更新依賴鎖檔並通過 dependency audit，修正 Windows CI 換行差異造成的假失敗。

## 安裝

從本版本 GitHub Release 下載 `xiangqi-analyzer-0.3.2-setup.exe`。全新安裝會建立桌面與開始功能表捷徑，首次啟動可直接測試內建 Pikafish；Gemini API Key 不包含在安裝檔或 GitHub，必須由使用者在 App 內自行輸入，並由 Windows `safeStorage` 加密保存在本機。

## 發行驗證

本版本必須通過 typecheck、針對性單元／整合測試、dependency audit、Windows NSIS 封裝、GitHub update metadata 與 SHA-512 驗證，再以從 GitHub Release 重新下載的安裝檔完成乾淨安裝及桌面 smoke test。

## 已知限制

v0.3.2 尚未配置受信任 CA 核發的 Windows 程式碼簽章，Windows SmartScreen 可能顯示警告。這需要外部簽章憑證才能解除；本專案不以自簽憑證冒充公開信任。
