# 象棋 AI 分析講解 v0.3.1

v0.3.1 是以實際桌面操作回饋為核心的體驗與可靠性修正版，重點是讓棋盤、AI 教練與即時引擎分析在同一個視窗內持續可見，不必捲動整個頁面。

## 主要改善

- 重做分析頁的視窗比例：工具列、棋盤／AI 工作區與底部即時分析固定分配可用高度；一般桌面視窗不再需要上下捲動頁面才能看到引擎結果。
- 棋盤會依視窗高度縮放，「放大／縮小棋盤」在低高度視窗也有明確差異；底部 Live 面板只顯示最新主線與雙引擎摘要，不再出現捲軸，完整候選線與原始資料集中在上方「分析資料」抽屜。
- 底部同時出現多個引擎限制時會合併成兩行內的單一摘要，完整警告仍保留在分析資料中，避免 1024×700 視窗裁掉雙引擎結果。
- 即時主引擎與複核引擎分析持續加深；AI 解說、切換檢視與自動解說不會形成無限重啟迴圈，使用者停止後也不會被後續 Live 結果擅自重新啟動。
- 棋盤加入完整鍵盤操作：方向鍵移動焦點，Enter／空白鍵操作格子，並提供格子、棋子與選取狀態的可讀標籤。
- 分析資料抽屜開啟時，背後棋盤與教練區不再被 Tab 或讀屏誤觸；可用 Escape 關閉並把焦點送回「分析資料」。AI 教練／猜著分頁也支援方向鍵、Home 與 End 的正確焦點移動。
- 即時引擎不再讓讀屏每秒重複播報耗時與主線；錯誤、一般通知與階段狀態使用各自正確的提示語意。
- 危險操作加入明確確認，設定、資料回復與錯誤訊息更容易理解。
- 本機資料檔讀取失敗時會保留原檔並封鎖後續寫入，避免以空白資料覆蓋損壞、過大或暫時無法讀取的資料；使用者可在畫面上重試。
- AI 解說在未提供「使用者著法」時改為分析當前局面，不再虛構使用者走法或錯誤；繁體中文、簡體中文與英文共用相同的有界品質流程。
- Gemini `generateContent` 請求使用端點實際支援的 `responseMimeType: application/json`，並降低不必要的思考預算；回應只擷取非 thought 的最終內容，減少格式失敗與等待時間。
- 同一對話的追問只呼叫一次模型、不重跑引擎研究，會遵守句數與格式要求；JSON 異常時直接使用發問當下的引擎快照產生精簡回答，不再重生整份教學。
- 英文追問的 `one`～`five sentences` 也會被辨識與驗證；取消恰好發生在錯誤回應本文讀取時，仍會正確顯示為取消而不是 Provider 錯誤。
- AI 對話固定保存發問當下的局面、引擎證據與 provider／model 來源；持久化資料會正規化無效的舊欄位，避免損壞資料使畫面崩潰。
- 更新產物驗證新增安裝檔大小、ProductVersion、`latest.yml` size 與封裝後更新 URL；網站發布會拒絕達到 100 MiB Git blob 上限的檔案。

## 安裝

下載 `xiangqi-analyzer-0.3.1-setup.exe`，可直接覆蓋既有安裝。使用者資料與透過 Electron `safeStorage` 加密的 API Key 保存在個人資料目錄，不包含在安裝檔內。

## 發行驗證

本版本只有在 typecheck、完整自動測試、dependency audit、production build、Windows NSIS 封裝、更新產物驗證，以及安裝版實際桌面操作全部通過後才發布。GitHub Release、公開 `latest.yml` 與安裝到桌機的執行檔必須是同一組 v0.3.1 產物。

## 已知限制

v0.3.1 安裝檔仍缺少受信任 CA 核發的 Windows 程式碼簽章；Windows SmartScreen 可能顯示警告。這是需要外部簽章憑證才能解除的發行阻塞，不以自簽憑證冒充公開信任。正式公開販售前仍須配置 `WINDOWS_CSC_LINK`／`WINDOWS_CSC_KEY_PASSWORD`，並重新驗證安裝與自動更新流程。

第三方象棋引擎二進位與 NNUE 權重不包含在 App 內，使用者仍須自行合法取得並在設定頁加入。
